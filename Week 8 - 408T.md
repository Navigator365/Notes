
To set up a basic backdoor, we hook on recv, invoking the original function and then parsing its buffer for an invocation of a command, which we then execute and send back through calling the original send function.  If we detect a command execution, we'll only return any content before the command invocation (or nothing if we only received a command). If we don't detect a command execution, we'll return all the content we received. 

Next, we'll hook on 5 functions to avoid detection: 
1. **fopen** - We hook here to prevent our file from being read by **more/less**/other related commands, which all use this function to open the file before displaying its contents. When we detect our file as a path argument to this command, we return null, and set an error code to signal that the file doesn't exist. 
2. **open** - We hook here to prevent our file from being read by **cat**, which uses this function to open the file before displaying its contents. When we detect our file as a path argument to this command, we return an error signaling that the file doesn't exist. 
3. **readdir** - We hook here to prevent our file and directory from appearing inside a directory when **ls** is invoked over it. When ls is invoked over a directory, this function will be called on every folder or file inside it, returning a struct. We call the original readdir function, then parse the returned struct for its name to see if it matches our file. If it does, we'll call readdir again, which returns another file. The files can be imagined existing in a linked-list structure: once we call one file, we won't call it again, causing it not to be displayed by ls.
4. **statx** - We hook here to prevent our file from appearing when **ls** is invoked over a named filepath (ie ls my_backdoor/backdoor.so). When we detect our file as a path argument to this command, we return an error signaling that the file doesn't exist. 
5. **unlinkat** - We hook here to prevent our file/directory from being removed when **rm** is called with it as an argument. When we detect our file as a path argument to this command, we return an error signaling that the file doesn't exist.
